## UA2 – Interface, interação e Affordance

### Objetivo
O objetivo desta atividade foi projetar uma interface que se adapte e se molde aos usuários com deficiência visual, garantindo que essa limitação não os impeça de utilizar a interface e realizar seu trabalho normalmente, conforme o cenário de um artefato gerenciador de cursos internos de uma empresa.

### Solução Desenvolvida
A solução desenvolvida focou na criação de uma interface inclusiva para funcionários da área de treinamento com deficiência visual, respondendo aos seguintes pontos:

**a) Sugestão de Interface:** Eu sugeriria uma interface gráfica acessível, que funcione bem com leitores de tela. Usuários cegos no geral já têm o costume de navegar pelo computador usando só o teclado, então a interface precisa ser desenvolvida pensando nisso. Com o Tab eles vão passando por todos os elementos da tela, com Enter confirmam, e com as setas navegam pelas opções. O leitor de tela vai lendo em voz alta cada botão e campo conforme o usuário navega, então desde que a interface esteja bem estruturada, ele consegue trabalhar normalmente sem depender do mouse ou da visão.

**b) Equipamento Necessário:** O equipamento seria um computador com teclado e fone de ouvido junto com um software leitor de tela. Os mais usados são o NVDA, que é gratuito, ele lê o conteúdo da tela em voz alta e são bastante utilizados por pessoas com deficiência visual no dia a dia.
